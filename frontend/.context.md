# Estate CRM Frontend - Context for AI Agents

## Project Overview

**Estate CRM** is a Real Estate Customer Relationship Management (CRM) system with a modern web frontend for managing properties, clients, deals, and meetings.

- **Framework**: React 19 + TypeScript + Vite
- **Styling**: TailwindCSS + Shadcn UI
- **State Management**: Zustand (for authentication)
- **Data Fetching**: TanStack React Query (React Query v5)
- **Routing**: React Router v7
- **HTTP Client**: Fetch API with custom wrapper

## Key Concepts

### 1. Authentication Flow

```
User Login → loginRequest() → Backend /auth/login (POST)
       ↓
   AuthResponse received (accessToken, refreshToken, user data)
       ↓
   Store in Zustand (useAuthStore) + localStorage
       ↓
   Redirect to /dashboard
```

**Auth Files**:

- `src/entities/auth/model/authStore.ts` - Zustand store (persists to localStorage)
- `src/entities/auth/model/useLogin.ts` - Login mutation hook
- `src/entities/auth/api/auth.ts` - Login API call
- `src/entities/auth/ui/LoginPageContent.tsx` - Login UI
- `src/app/router/ProtectedRoute.tsx` - Route guard
- `src/app/router/AuthRoute.tsx` - Auth-only redirect

**Auth Store Structure**:

```typescript
interface AuthState {
  accessToken: string | null
  refreshToken: string | null
  userId: number | null
  fullName: string | null
  email: string | null
  role: 'AGENT' | 'ADMIN'
  setAuth: (payload) => void
  setTokens: (accessToken, refreshToken) => void
  setUser: (user) => void
  clearAuth: () => void
}
```

### 2. HTTP Client

**File**: `src/shared/api/base.ts`

```typescript
// Automatically adds:
// - Authorization: Bearer <token>
// - Content-Type: application/json
// - Uses VITE_API_URL environment variable

const api = {
  get: (url) => request(url, { method: 'GET' }),
  post: (url, body) =>
    request(url, { method: 'POST', body: JSON.stringify(body) }),
  put: (url, body) =>
    request(url, { method: 'PUT', body: JSON.stringify(body) }),
  delete: (url) => request(url, { method: 'DELETE' })
}
```

**Usage**:

```typescript
const data = await api.get<T>('/endpoint')
```

### 3. Global Providers (AppProvider.tsx)

```
AppProviders wraps:
├── QueryClientProvider (React Query)
│   └── defaultOptions: staleTime=30s, retry=1, refetchOnWindowFocus=false
├── ThemeProvider (dark theme by default, key="vite-ui-theme")
├── TooltipProvider (Shadcn)
└── ReactQueryDevtools (dev only)
```

### 4. Routing Structure

```
/                      → Redirects to /dashboard
/login                 → LoginPage (public, AuthRoute guards)
/dashboard             → DashboardPage (protected)
/clients               → ClientsPage (protected)
/properties            → PropertiesPage (protected)
/deals                 → DealsPage (protected)
/meetings              → MeetingsPage (protected)
/upcoming              → UpcomingPage (protected)
```

**Route Protection**:

- `AuthRoute`: Only allows non-authenticated users
- `ProtectedRoute`: Checks accessToken, redirects unauthenticated users to /login

## Entity Structure Pattern

Each entity (clients, properties, deals, etc.) follows:

```
entity/

│   ├── index.ts              # Export all API functions
```

## Data Fetching with React Query

### Key Configuration

```typescript
// Defaults in src/app/providers/AppProvider.tsx
const queryClient = new QueryClient({
  defaultOptions: {
    queries: {
      staleTime: 30_000, // 30 seconds
      retry: 1, // Retry failed requests once
      refetchOnWindowFocus: false // Don't refetch unless explicitly needed
    }
  }
})
```

### Usage Pattern

```typescript
// In hooks
import { useQuery, useMutation } from '@tanstack/react-query'

// Queries (GET)
const { data, isPending, error } = useQuery({
  queryKey: ['clients'],
  queryFn: () => api.get('/clients')
})

// Mutations (POST/PUT/DELETE)
const { mutate, isPending } = useMutation({
  mutationFn: (data) => api.post('/clients', data),
  onSuccess: () => {
    queryClient.invalidateQueries({ queryKey: ['clients'] })
  }
})
```

## Common Development Tasks

### Adding a New Entity

1. Create `src/entities/[entity]/api/[entity].ts` - REST calls
2. Create `src/entities/[entity]/model/types.ts` - TypeScript types
3. Create `src/entities/[entity]/model/use[Entity].ts` - Query hooks
4. Create `src/entities/[entity]/ui/` - Components if needed
5. Import and use in pages

### Adding a New Page

1. Create `src/pages/[NewPage].tsx`
2. Add route to `src/app/router/AppRouter.tsx`
3. Add navigation link in `src/components/AppSidebar.tsx`

### Using React Query

```typescript
// Always inside a component wrapped with AppProviders
const { data, isPending, error } = useQuery({
  queryKey: ['entity', id],
  queryFn: () => api.get(`/entity/${id}`)
})
```

## Important Notes for AI Agents

1. **Always use `api` wrapper** from `src/shared/api/base.ts` for HTTP calls - it handles auth tokens automatically
2. **Zustand stores are persisted** - Check localStorage if debugging auth issues
3. **React Query DevTools** available in dev mode (bottom right corner)
4. **Never fetch directly** - Use React Query hooks for server state
5. **TailwindCSS** - All styling should use utility classes, not separate CSS files
6. **TypeScript strict** - Ensure proper typing for components and props
7. **Dark theme** - Design with dark mode in mind (zinc-950, zinc-900, etc.)
8. **Protected routes** - All pages (except /login) are wrapped in ProtectedRoute
9. **API URL** - Set via VITE_API_URL environment variable
10. **Authentication token** - Automatically added to all requests via base.ts wrapper

## Backend Integration

### API Endpoints Used

- `POST /auth/login` - User login
- `POST /auth/register` - User registration
- `POST /auth/refresh` - Refresh access token
- `GET /auth/me` - Get current user
- `GET /clients` - List all clients
- `POST /clients` - Create client
- `GET /properties` - List all properties
- `POST /properties` - Create property
- `GET /deals` - List all deals
- `POST /deals` - Create deal
- `GET /meetings` - List all meetings
- `POST /meetings` - Create meeting
- `GET /dashboard` - Dashboard stats

### Error Handling

```typescript
// Backend returns 401 for unauthorized
// base.ts wrapper logs error, can add refresh token logic
if (res.status === 401) {
```
